  • ReadBuffer

    Declaration

    class ReadBuffer: dlib.container.buffer.Buffer;

    Self-expanding buffer, that can be used with functions returning the number of the read bytes.

    Discussion

    This buffer supports asynchronous reading. It means you can pass a new chunk to an asynchronous read function during you are working with already available data. But only one asynchronous call at a time is supported. Be sure to call ReadBuffer.clear() before you append the result of the pended asynchronous call.

  • WriteBuffer

    Declaration

    class WriteBuffer: dlib.container.buffer.Buffer;

    Circular, self-expanding buffer with overflow support. Can be used with functions returning returning the number of the transferred bytes.

    Discussion

    The buffer is optimized for situations where you read all the data from it at once (without writing to it occasionally). It can become ineffective if you permanently keep some data in the buffer and alternate writing and reading, because it may allocate and move elements.

    • opSlice

      Declaration

      pure nothrow @nogc @property @safe ubyte[] opSlice(size_t start, size_t end);

      Returns a chunk with data.

      Discussion

      After calling it, set += to the length could be written.

      buffer may return only part of the data. You may need to call it (and set += several times until length is 0). If all the data can be written, maximally 3 calls are required.

      Return Value

      A chunk of data buffer.

      Examples

      1. auto b = defaultAllocator.make!WriteBuffer(6); ubyte[6] buf = [23, 23, 255, 128, 127, 9]; b ~= buf; assert(b[0..$] == buf[0..6]); b += 2; assert(b[0..$] == buf[2..6]); b ~= buf; assert(b[0..$] == buf[2..6]); b += b.length; assert(b[0..$] == buf[0..6]); b += b.length; defaultAllocator.dispose(b);
